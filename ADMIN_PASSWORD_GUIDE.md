# 🔐 ميزة تغيير كلمة المرور - دليل المسؤول

## 📋 ملخص

تم إضافة ميزة قوية وآمنة تسمح للمسؤول بتغيير كلمات مرور المستخدمين مباشرة من لوحة التحكم.

---

## ✨ الميزات

### 1️⃣ إنشاء كلمة مرور مؤقتة
```
الاستخدام: للمستخدمين الجدد أو الذين نسوا كلمتهم
كيفية العمل:
  1. اضغط "إنشاء كلمة مرور مؤقتة"
  2. الخادم ينشئ كلمة عشوائية آمنة
  3. انسخها وأرسلها للمستخدم
  4. المستخدم يغيرها عند أول دخول

المثال: Temp5847293
الفائدة: سهل وآمن، لا تحتاج لاختيار كلمة
```

### 2️⃣ تعيين كلمة مرور محددة
```
الاستخدام: عندما تريد كلمة محددة معينة
كيفية العمل:
  1. أدخل كلمة المرور الجديدة
  2. أعد إدخالها للتأكيد
  3. اضغط تحديث
  4. تطبق فوراً

الشروط:
  - 6 أحرف على الأقل
  - يجب تطابق الحقلين
  - يفضل أن تكون قوية

الفائدة: تحكم كامل على كلمة المرور
```

## 🛠️ المحتويات المضافة

### Backend

#### الـ Endpoints:

**1. تغيير كلمة المرور (Custom Password)**
```javascript
PUT /api/users/:userId/password
Authorization: Bearer {admin-token}

Request Body:
{
  "newPassword": "MyNewPassword123"
}

Response (Success):
{
  "message": "تم تحديث كلمة المرور بنجاح"
}

Response (Error):
{
  "error": "كلمة المرور يجب أن تكون 6 أحرف على الأقل"
}
```

**2. إنشاء كلمة مرور مؤقتة**
```javascript
POST /api/users/:userId/reset-password
Authorization: Bearer {admin-token}

Request: (No body needed)

Response (Success):
{
  "message": "تم إعادة تعيين كلمة المرور",
  "tempPassword": "Temp5847293",
  "note": "تأكد من إرسال كلمة المرور المؤقتة للمستخدم"
}

Response (Error):
{
  "error": "المستخدم غير موجود"
}
```

#### الـ API Methods:

**api.ts** (محدث)
```typescript
// تغيير كلمة المرور
async changeUserPassword(userId: string, newPassword: string) {
  return this.request<any>(`/users/${userId}/password`, {
    method: 'PUT',
    body: { newPassword }
  });
}

// إعادة تعيين كلمة المرور
async resetUserPassword(userId: string) {
  return this.request<any>(`/users/${userId}/reset-password`, {
    method: 'POST'
  });
}

الملف: src/lib/api.ts
```

## 📊 حالات الاستخدام

### مثال 1: مستخدم جديد

```
السيناريو: تم إضافة مستخدم جديد "محمد علي"

الخطوات:
1. المسؤول يفتح صفحة المستخدم
2. ينقر على ثلاث نقاط بجانب محمد علي
3. يختار "تغيير كلمة المرور"
4. ينقر "إنشاء كلمة مرور مؤقتة"
5. الخادم ينشئ: "Temp7392847"
6. المسؤول ينسخها ويرسلها لمحمد:
   "رحباً محمد، بيانات الدخول:
    البريد: m.ali@hospital.com
    كلمة المرور: Temp7392847"
7. محمد يسجل دخول بالكلمة المؤقتة
8. النظام يطلب منه تغيير الكلمة
9. محمد يختار كلمة جديدة تاركة من اختياره
```

### مثال 2: كلمة مرور منسية

```
السيناريو: فاطمة نسيت كلمة المرور

الخطوات:
1. فاطمة تتصل بالمسؤول: "نسيت كلمتي"
2. المسؤول يفتح صفحة فاطمة
3. يختار "تغيير كلمة المرور"
4. ينقر "إنشاء كلمة مرور مؤقتة"
5. النظام ينشئ: "Temp4829473"
6. المسؤول يرسل لفاطمة الكلمة
7. فاطمة تسجل دخول بها
8. تغير إلى كلمة اختيارها
```

### مثال 3: سياسة أمان

```
السيناريو: تحديث كلمات المرور دورياً

الخطوات:
1. كل 3 أشهر، المسؤول يراجع قائمة المستخدمين
2. لكل مستخدم:
   - يختار "تغيير كلمة المرور"
   - يختار "تعيين كلمة محددة" قوية
   - يرسل الكلمة الجديدة
3. المستخدم يسجل دخول بالكلمة الجديدة
4. يفعل الخطوة المطلوبة لتحديثها
```

## ❌ معالجة الأخطاء

### الخطأ 1: كلمة المرور قصيرة جداً

```
الرسالة: "كلمة المرور يجب أن تكون 6 أحرف على الأقل"
السبب: أدخلت أقل من 6 أحرف
الحل: أدخل 6 أحرف أو أكثر

مثال خاطئ: "Pass1" (5 أحرف)
مثال صحيح: "Pass123" (7 أحرف)
```

### الخطأ 2: عدم تطابق كلمات المرور

```
الرسالة: "كلمات المرور غير متطابقة"
السبب: الحقلان لا يحتويان نفس الكلمة
الحل: أعد كتابة الكلمة بنفس الطريقة

مثال خاطئ:
  كلمة المرور: "MyPassword123"
  تأكيد: "MyPassword321"

مثال صحيح:
  كلمة المرور: "MyPassword123"
  تأكيد: "MyPassword123"
```

### الخطأ 3: المستخدم غير موجود

```
الرسالة: "المستخدم غير موجود"
السبب: المستخدم قد تم حذفه من النظام
الحل: تحقق من أن المستخدم موجود في القائمة
```

### الخطأ 4: فشل الاتصال

```
الرسالة: "فشل تحديث كلمة المرور"
السبب: مشكلة في الاتصال بالخادم
الحل:
  1. تحقق من اتصالك بالإنترنت
  2. تأكد من تشغيل Backend
  3. جرب مرة أخرى
```
