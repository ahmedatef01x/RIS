# ميزة تعديل بيانات المريض

## الملخص
تم إضافة قابلية **تعديل بيانات المريض** من قائمة **آخر المرضى** في صفحة التسجيل.

## المميزات الجديدة

### 1. قائمة المرضى الأخيرين قابلة للنقر
- عند النقر على أي مريض في قائمة "آخر المرضى"، يتم تحميل بيانته في النموذج
- المريض المختار يظهر بخلفية مزرقة مع حدود قوية للدلالة على أنه قيد التعديل
- رسالة "تحرير في التقدم..." تظهر تحت المريض المختار

### 2. تحميل بيانات المريض (`loadPatientForEdit`)
```typescript
loadPatientForEdit(patientId: string)
```
- تحميل جميع بيانات المريض من قاعدة البيانات
- ملء جميع حقول النموذج بالبيانات الموجودة
- تحديد `editingPatientId` لتفعيل وضع التعديل
- عرض رسالة توفيق "تم تحميل بيانات المريض"

### 3. دعم الإنشاء والتحديث في `handleSubmit`
```typescript
if (editingPatientId) {
  // تحديث المريض الموجود
  await api.updatePatient(editingPatientId, patientData);
  toast.success("تم تحديث بيانات المريض بنجاح!");
} else {
  // إنشاء مريض جديد
  await api.createPatient(patientData);
  toast.success("تم تسجيل المريض بنجاح!");
}
```

### 4. تغيير نص الزر
- عند التعديل: "تحديث المريض"
- عند الإنشاء الجديد: "تسجيل المريض"

### 5. تحديث عنوان النموذج
- عند التعديل: "تحرير بيانات المريض"
- عند الإنشاء الجديد: "البيانات الشخصية"

### 6. دالة إلغاء الاختيار (`handleClearForm`)
- تصفير جميع حقول النموذج
- إلغاء وضع التعديل
- إعادة تعيين الاقتراحات

## تدفق العمل

### إنشاء مريض جديد
1. الصفحة تظهر "مريض جديد" بشكل افتراضي
2. ملء البيانات
3. النقر على "تسجيل المريض"
4. حفظ في قاعدة البيانات
5. تحديث قائمة "آخر المرضى"

### تعديل بيانات مريض موجود
1. في قسم "آخر المرضى"، النقر على المريض المطلوب تعديله
2. تحميل بيانات المريض في النموذج تلقائياً
3. تعديل البيانات المطلوبة
4. النقر على "تحديث المريض"
5. حفظ التغييرات في قاعدة البيانات
6. تحديث قائمة "آخر المرضى"

### البحث والتعديل
1. استخدام تبويب "بحث عن مريض" للبحث
2. النقر على أي نتيجة بحث لتعديل بياناتها
3. تحرير البيانات
4. النقر على "تحديث المريض"

## التكامل مع الواجهة

### الحقول المدعومة
- رقم الملف (MRN)
- الأسماء (4 أجزاء عربي + إنجليزي)
- العمر والنوع
- رقم الهاتف
- العنوان
- رقم التأمين
- جهة التحويل
- التاريخ المرضي

### المميزات المحفوظة
- قاموس الأسماء يعمل بكامل قوته
- الاقتراحات التلقائية للأسماء
- إضافة أسماء جديدة إلى القاموس
- التحقق من الحقول المطلوبة

## التغييرات المضافة

### حالات جديدة
```typescript
const [editingPatientId, setEditingPatientId] = useState<string | null>(null);
```

### دوال جديدة
1. `loadPatientForEdit(patientId)` - تحميل بيانات المريض
2. `handleClearForm()` - تصفير النموذج

### تعديلات على الدوال الموجودة
1. `handleSubmit()` - دعم التحديث والإنشاء
2. UI تبويب النموذج - عنوان ديناميكي
3. قائمة المرضى - نقر قابل للاختيار

## التوافق

### قاعدة البيانات
- يستخدم نفس جداول وأعمدة المريض الموجودة
- دعم كامل للحقول الثمانية للأسماء المنقسمة

### API
- `GET /api/patients/:id` - للحصول على بيانات المريض
- `PUT /api/patients/:id` - لتحديث بيانات المريض
- `POST /api/patients` - لإنشاء مريض جديد (لم يتغير)

### Supabase
- التحديث يعمل مع Supabase أيضاً
- دعم كامل للعمليات المحلية والسحابية

## ملاحظات مهمة

1. **الحقول المطلوبة**: لا تزال متطلبة (الاسم الأول والهاتف)
2. **MRN**: لا يمكن تغييره عند التحديث (يتم حفظه فقط)
3. **القاموس**: الأسماء المضافة جديداً تُحفظ في قاعدة البيانات المشتركة
4. **الإعادة**: بعد التعديل الناجح، يتم تحديث قائمة المرضى تلقائياً
