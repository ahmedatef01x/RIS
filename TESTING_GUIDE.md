### 1. افتح المتصفح وانتقل إلى التطبيق:
```
http://localhost:5174
```

### 2. ستظهر شاشة تسجيل الدخول (Auth Page)

### 3. ادخل بيانات دخول الإدارة:
- **البريد الإلكتروني:** `admin@ris.com`
- **كلمة المرور:** `12345678`

### 4. اضغط على زر "تسجيل الدخول" (Sign In)

---

## النتائج المتوقعة:

✅ **السيناريو الناجح:**
1. بعد الضغط على زر تسجيل الدخول مباشرة، ستنتقل إلى شاشة "جاري التحميل" (Loading Screen)
2. سترى spinner يدور مع نص "جاري التحميل..." و"جاري جلب تفضيلاتك"
3. **خلال أقل من ثانية واحدة** (عادة ما يكون 300-500ms)، سيتم إعادة توجيهك تلقائياً إلى **لوحة البيانات** (Dashboard)
4. ستتمكن من رؤية جميع الصفحات في القائمة الجانبية (Sidebar):
   - 📊 لوحة البيانات
   - 👥 المرضى
   - 📋 قائمة الانتظار
   - 📅 الجدولة
   - 📄 التقارير
   - 💰 الفواتير
   - 🔔 الإشعارات
   - 👤 المستخدمون
   - 🖥️ الأجهزة
   - 📸 أنواع الفحوصات
   - ⚙️ الإعدادات
   - 🔐 لوحة التحكم (Admin Panel)

## حالات الأخطاء المحتملة والحلول:

### ❌ إذا ظلت شاشة "جاري التحميل" معلقة لأكثر من 10 ثواني:

**الخطوة 1: افتح Developer Tools**
- اضغط على `F12` أو `Ctrl+Shift+I` 
- اذهب إلى تبويب `Network`

**الخطوة 2: تحقق من الطلبات**
- ابحث عن طلب يسمى `preferences` أو `/api/users/.../preferences`
- تحقق من:
  - هل الطلب مكتمل؟
  - ما هو كود الحالة (Status Code)؟ يجب أن يكون `200`
  - كم وقت استغرق الطلب؟

**الخطوة 3: تحقق من رسالة الخطأ**
- في Console (تبويب Console):
  - هل هناك رسالة خطأ حمراء؟
  - هل رسالة "❌ Preferences fetch timeout"؟

**الخطوة 4: الحل**
- إذا كان الطلب يستغرق وقتاً طويلاً:
  1. تأكد من أن الـ Backend يعمل على `http://localhost:3001`
  2. اختبر اتصال البيانات بفتح نافذة محرر جديدة والذهاب إلى:
     ```
     http://localhost:3001/api/users/4D6F743A-C744-4D7A-9D75-4C7FF16306AB/preferences
     ```
     (مع وضع Authorization header يحتوي على JWT token من browser local storage)

---

### ❌ إذا ظهرت رسالة خطأ "حدث خطأ. جاري الإعادة...":

**المعنى:** هناك مشكلة في جلب البيانات من الـ Backend

**الحلول:**
1. **تحقق من الـ Backend Status:**
   ```
   curl http://localhost:3001/health
   ```
   إذا لم تحصل على رد، فالـ Backend قد لا يكون يعمل

2. **أعد تشغيل الـ Backend:**
   ```bash
   cd local-backend
   npm start
   ```

3. **تحقق من قاعدة البيانات:**
   ```bash
   cd local-backend
   node check-user.js
   ```
   يجب أن تري:
   ```
   ✅ User found
   ✅ Preferences found
   ✅ Permissions found
   ```

---

### ❌ إذا دخلت إلى Dashboard لكن الصفحات الأخرى تظهر "لا توجد صلاحيات":

**المعنى:** المستخدم قد لا يكون لديه صلاحيات مناسبة

**الحل:**
```bash
cd local-backend
node init-permissions.js
```

ثم أعد تحديث الصفحة (F5)

## اختبارات إضافية:

### ✅ اختبر الانتقال بين الصفحات:
1. من Dashboard، اضغط على "المرضى" (Patients)

3. ستعود إلى شاشة تسجيل الدخول
4. حاول تسجيل الدخول مرة أخرى - يجب أن تعود إلى Dashboard مباشرة

اضغط على كل صفحة من الصفحات التالية وتأكد من:
- تحميل الصفحة بسرعة
- عدم وجود أخطاء في Console
- ظهور البيانات بشكل صحيح

## معلومات تقنية للتصحيح:

**مسار الملفات المحدثة:**
- `src/components/HomeRedirect.tsx` - مكون التوجيه مع timeout
- `local-backend/src/routes/users.js` - API endpoint للتفضيلات
- `src/lib/api.ts` - API client مع timeout (لم يتغير)

**البيانات المختبرة:**
- User ID: `4D6F743A-C744-4D7A-9D75-4C7FF16306AB`
- Email: `admin@ris.com`
- Role: `admin`
- Default Homepage: `dashboard`
- Permissions: 11 صفحات × 4 صلاحيات = 44 صلاحية

**المدة المتوقعة:**
- تسجيل الدخول → تحميل التفضيلات: 300-500ms
- الإعادة التلقائية إلى Dashboard: تلقائية عند انتهاء التحميل
- Timeout (إذا فشل): 10 ثواني ثم إعادة توجيه

## نصائح للأداء:

1. **استخدم DevTools Network Throttling:**
   - اختبر مع `Slow 3G` أو `4G` للتأكد من التعامل مع البطء
   - يجب أن تظهر رسالة التحذير بعد 5 ثواني

2. **استخدم DevTools Console:**
   - ابحث عن أيقونات 🏠 و 🔍 و ❌ للتتبع
   - كل واحدة تخبرك بالمرحلة الحالية

3. **تحقق من Local Storage:**
   - افتح DevTools → Application
   - اذهب إلى Local Storage → localhost:5174
   - تحقق من وجود مفتاح `token`
