# 🔐 ميزة تغيير كلمة المرور - دليل استخدام

## ✨ ما الجديد؟

تم إضافة ميزة تسمح للمسؤول (Admin) بتغيير كلمة مرور المستخدمين بسهولة.

---

## 🚀 كيفية الاستخدام

### الخطوة 1: اذهب إلى صفحة إدارة المستخدمين
- من القائمة الجانبية → لوحة التحكم → إدارة المستخدمين
- أو اضغط على "Admin Panel" → "User Management"

### الخطوة 2: اختر المستخدم
- ابحث عن المستخدم في القائمة
- اضغط على الثلاث نقاط (⋯) في نهاية صف المستخدم

### الخطوة 3: اختر "تغيير كلمة المرور"
من القائمة المنسدلة:
```
Edit              ✎
تغيير كلمة المرور 🔐  ← اضغط هنا
إدارة الصلاحيات   🔒
تعطيل/تفعيل       👤
```

### الخطوة 4: اختر الخيار المناسب

#### الخيار 1️⃣: إنشاء كلمة مرور مؤقتة
```
✨ الميزة: الخادم ينشئ كلمة مرور عشوائية آمنة
📋 الخطوات:
   1. اضغط "إنشاء كلمة مرور مؤقتة"
   2. سيظهر الرمز مثل: Temp5847293
   3. انسخ الرمز (اضغط الزر Copy)
   4. أرسل الرمز للمستخدم بشكل آمن
   
⚠️ ملاحظة: كلمة المرور تكون عشوائية وآمنة تماماً
```

#### الخيار 2️⃣: تعيين كلمة مرور محددة
```
✨ الميزة: تحديد كلمة مرور خاصة بك
📋 الخطوات:
   1. ادخل كلمة المرور الجديدة (6+ أحرف)
   2. أعد إدخال كلمة المرور للتأكيد
   3. تحقق من تطابق كلمات المرور
   4. اضغط "تحديث كلمة المرور"

⚠️ شروط كلمة المرور:
   - 6 أحرف على الأقل
   - يجب أن تكون متطابقة في الحقلين
   - يفضل أن تكون قوية (أحرف + أرقام + رموز)
```

## 🎯 الحالات الاستخدام

### الحالة 1: المستخدم نسي كلمة المرور
```
✅ الحل: استخدم الخيار 1 (كلمة مرور مؤقتة)
   1. أنشئ كلمة مرور مؤقتة
   2. أرسلها للمستخدم عبر البريد
   3. يمكن للمستخدم تغييرها لاحقاً
```

### الحالة 2: كلمة المرور ضعيفة
```
✅ الحل: استخدم الخيار 2 (تعيين كلمة محددة)
   1. عيّن كلمة مرور قوية
   2. أرسلها للمستخدم بشكل آمن
   3. أخبره بضرورة تغييرها عند أول دخول
```

### الحالة 3: مستخدم جديد
```
✅ الحل: استخدم الخيار 1 (كلمة مرور مؤقتة)
   1. أنشئ كلمة مرور مؤقتة
   2. اعطه بيانات الدخول
   3. يغير كلمة المرور عند أول دخول
```

## 🔒 الأمان

### ✅ معايير الأمان المطبقة:

1. **التشفير القوي (bcrypt):**
   - كل كلمة مرور تُشفّر قبل التخزين
   - لا يمكن فك تشفيرها

2. **التحقق من الصلاحيات:**
   - فقط المسؤول (Admin) يمكنه تغيير كلمات المرور
   - كل تغيير يتم تسجيله

3. **HTTPS:**
   - جميع الطلبات محمية بـ HTTPS
   - لا يمكن اعتراض البيانات

4. **التسجيل:**
   - كل تغيير يتم تسجيله مع معلومات المسؤول

## 🛠️ التفاصيل التقنية

### الـ Endpoints الجديدة:

#### 1. تغيير كلمة المرور (Custom)
```
PUT /api/users/:userId/password
Authorization: Bearer {token}
Body: {
  "newPassword": "NewPassword123"
}

Response: {
  "message": "تم تحديث كلمة المرور بنجاح"
}
```

#### 2. إعادة تعيين كلمة المرور (Temporary)
```
POST /api/users/:userId/reset-password
Authorization: Bearer {token}

Response: {
  "message": "تم إعادة تعيين كلمة المرور",
  "tempPassword": "Temp5847293",
  "note": "تأكد من إرسال كلمة المرور المؤقتة للمستخدم"
}
```

### الـ Components:
- `ChangePasswordDialog.tsx` - مكون الـ dialog
- `AdminUserManagement.tsx` - صفحة إدارة المستخدمين

### الـ API Methods:
- `api.changeUserPassword(userId, password)` - تغيير مباشر
- `api.resetUserPassword(userId)` - توليد كلمة مؤقتة

## 🎓 أمثلة عملية

### مثال 1: مستخدم جديد

```
المسؤول:
  1. ينشئ مستخدم جديد: "أحمد علي"
  2. يفتح "تغيير كلمة المرور"
  3. ينقر "إنشاء كلمة مرور مؤقتة"
  4. ينسخ: "Temp9283474"
  5. يرسل البريد: "رحباً أحمد، كلمة المرور المؤقتة: Temp9283474"

المستخدم:
  1. يتلقى البريد
  2. يسجل دخول مع البيانات:
     - البريد: ahmed@hospital.com
     - كلمة المرور: Temp9283474
  3. يُطلب منه تغيير كلمة المرور
  4. يغير إلى كلمة المرور النهائية
```

### مثال 2: كلمة مرور مفقودة

```
المستخدم:
  - نسي كلمة المرور

المسؤول:
  1. يفتح صفحة المستخدم
  2. ينقر "تغيير كلمة المرور"
  3. ينقر "إنشاء كلمة مرور مؤقتة"
  4. ينسخ الكود وينسخه للمستخدم بشكل آمن

المستخدم:
  - يسجل دخول بالكود المؤقت
  - يغير كلمة المرور إلى كلمة اختياره
```
