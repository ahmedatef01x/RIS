# 🎯 ما العمل الآن؟

## الخطوة 1: تأكد من تشغيل الخادم ✅
```bash
# في نافذة Terminal منفصلة:
cd local-backend
npm start
```

يجب أن تظهر رسالة مثل:
```
✅ Connected to SQL Server
🔧 Initializing database schema...
✅ RIS Backend running on port 3001
```

## الخطوة 2: تأكد من تشغيل التطبيق ✅
```bash
# في نافذة Terminal أخرى:
npm run dev
```

يجب أن تظهر رسالة مثل:
```
VITE v5.4.19 ready in XXX ms

➜  Local:   http://localhost:5174/
```

## الخطوة 3: اختبر التطبيق ✅

1. **افتح المتصفح:**
   ```
   http://localhost:5174
   ```

2. **ستظهر شاشة تسجيل الدخول (Auth Page)**

3. **أدخل البيانات:**
   - البريد: `admin@ris.com`
   - كلمة المرور: `12345678`

4. **اضغط "تسجيل الدخول" (Sign In)**

## النتائج المتوقعة:

### ✅ النجاح:
```
Step 1: شاشة تسجيل دخول (Auth Page)
Step 2: شاشة تحميل (Loading Screen) مع spinner
        - "جاري التحميل..."
        - "جاري جلب تفضيلاتك"
Step 3: (خلال <1 ثانية) إعادة توجيه إلى Dashboard
Step 4: ترى لوحة البيانات مع الصفحات الجانبية
```

### ❌ إذا حدث خطأ:
```
Step 1: شاشة تحميل
Step 2: (بعد 5 ثواني) رسالة تحذير:
        "استغرق التحميل وقتاً طويلاً..."
Step 3: (بعد 10 ثواني) رسالة خطأ:
        "انتهت مهلة الانتظار..."
Step 4: (بعد ثانيتين) إعادة توجيه تلقائية إلى Dashboard
```

في كلا الحالتين، لن تبقى عالقاً على شاشة التحميل!

## الخطوة 4: تحقق من البيانات (اختياري)

إذا أردت التحقق من أن كل شيء صحيح:

```bash
cd local-backend

# تحقق من بيانات المستخدم
node check-user.js

# تحقق من دور المستخدم
node check-role.js

# أنشئ صلاحيات جديدة إذا احتجت
node init-permissions.js
```

بعد الدخول بنجاح، اختبر الصفحات التالية من الشريط الجانبي:

- ✅ لوحة البيانات (Dashboard)

- ✅ قائمة الانتظار (Worklist)
- ✅ الجدولة (Scheduling)
- ✅ التقارير (Reports)
- ✅ الفواتير (Billing)
- ✅ الإشعارات (Notifications)
- ✅ المستخدمون (Users)
- ✅ الأجهزة (Devices)
- ✅ أنواع الفحوصات (Exam Types)
- ✅ الإعدادات (Settings)
- ✅ لوحة التحكم (Admin Panel)

## 📞 حل المشاكل

### مشكلة: "لا يمكن الاتصال بـ Backend"
```bash
# تأكد من تشغيل:
cd local-backend && npm start
```

### مشكلة: "خطأ في قاعدة البيانات"
```bash
# تحقق من إعدادات .env:
cd local-backend
cat .env

# يجب أن تكون:
# DB_SERVER=127.0.0.1\SQLEXPRESS
# DB_USER=sa
# DB_PASSWORD=123456
```

### مشكلة: "لا توجد صلاحيات"
```bash
# أنشئ الصلاحيات:
cd local-backend
node init-permissions.js
```

### مشكلة: "المستخدم غير موجود"
```bash
# تحقق من:
node check-user.js

# إذا لم يكن موجوداً، استخدم صفحة التسجيل (Sign Up)
```

## 🎓 فهم العملية

### تسلسل تسجيل الدخول:
1. المستخدم يدخل البريد وكلمة المرور
2. يضغط "تسجيل الدخول"
3. الـ Frontend يرسل الطلب إلى `/auth/signin`
4. الـ Backend يتحقق من البيانات وينشئ JWT token
5. الـ Frontend يخزن الـ token في LocalStorage
6. يتم التوجيه إلى `/` (الصفحة الرئيسية)
7. `HomeRedirect` component يحمّل تفضيلات المستخدم
8. يعيد التوجيه إلى الصفحة الافتراضية (عادة Dashboard)

### مدة العملية:
- تسجيل الدخول: 100-200ms
- جلب التفضيلات: 300-500ms
- إعادة التوجيه: فوري
- **المجموع: <1 ثانية عادة**

## ✨ نصائح الأداء

1. **استخدم DevTools Network:**
   - اضغط F12
   - اذهب إلى Network
   - حدّد بطاقة `preferences`
   - شاهد وقت التحميل

2. **استخدم DevTools Console:**
   - اضغط F12
   - اذهب إلى Console
   - ابحث عن الرسائل بـ 🏠 و 🔍 و ❌

3. **اختبر مع Throttling:**
   - في Network، اختر "Slow 3G"
   - يجب أن تظهر رسالة التحذير بعد 5 ثواني
